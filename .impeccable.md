## Design Context

### Users
Students and academics evaluating DAT (Digital Asset Treasury) companies. The primary audience is course reviewers, professors, and classmates assessing the quality of data collection, visualization, and analysis. The dashboard should look credible enough that a real investor could use it, even though its primary purpose is academic demonstration.

### Brand Personality
**Modern, approachable, smart.** The interface should feel like a well-designed fintech product, not a homework assignment. It should communicate competence and clarity without being intimidating. Bloomberg Terminal is the aspirational reference for information density, but with friendlier aesthetics.

### Aesthetic Direction
- **Visual tone:** Dark financial dashboard. High contrast for data readability. Neon accent colors (teal, red, gold) for financial signals.
- **Reference:** Bloomberg Terminal for density and professionalism. CoinGecko for accessibility.
- **Anti-reference:** Generic SaaS templates, purple gradients, centered marketing layouts, bubbly UI.
- **Theme:** Dark mode only. Deep navy background (#0a0e17), muted card surfaces (#1a1f2e).
- **Typography:** Geist (sans) for UI, Geist Mono for financial data. No decorative fonts.
- **Color semantics:** Teal (#64ffda) = positive/premium. Red (#e94560) = negative/discount. Gold (#ffd93d) = highlights/warnings. Per-company colors for chart lines.

### Design Principles
1. **Data first.** Every pixel serves information. No decorative elements, blobs, or filler content. If a section feels empty, it needs better data, not decoration.
2. **Instant legibility.** mNAV values, trends, and company comparisons should be readable at a glance. Use color, size, and position to create clear hierarchy.
3. **Financial credibility.** The dashboard should look like it belongs on a trader's screen. Professional charting (TradingView Lightweight Charts), proper number formatting, consistent color semantics.
4. **Approachable complexity.** Dense data presented clearly. Tabs keep things organized. Cards summarize before charts detail. The AI chatbot makes data conversational.
5. **Responsive without compromise.** Mobile users should get a real dashboard experience, not a degraded desktop layout. 2-column cards on mobile, scrollable tabs, touch-friendly targets (44px minimum).
